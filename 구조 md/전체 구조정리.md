# 전체 구조정리

## 1. 프로젝트 한눈에 보기
- 형태: MSA 기반 가계부 SaaS
- 핵심 서비스
  - `auth-service`: 인증/회원/구독(Stripe)
  - `ledger-service`: 거래/카테고리/예산/대시보드/OCR 사용량
  - `ocr-ai-service`: OCR + 영수증 분류
- 데이터 저장소: MySQL
- 로컬 실행: `docker-compose.yml`

## 2. 현재 로컬 구조 (docker-compose)
- `auth-service`: `localhost:8001`
- `ledger-service`: `localhost:8002`
- `ocr-ai-service`: `localhost:8003`
- `mysql`: `localhost:3307`

요청 흐름
1. Frontend -> auth/ledger API 호출
2. ledger -> ocr-ai 내부 호출(`POST /ocr/classify`)
3. auth/ledger -> MySQL 접근
4. Stripe -> auth webhook(`POST /billing/webhook`)

## 3. 서비스별 책임
### 3.1 auth-service
- 회원가입/로그인/JWT 발급
- 내 정보 조회/수정/탈퇴
- Stripe checkout/session sync/cancel/webhook

### 3.2 ledger-service
- 거래 CRUD, CSV 내보내기
- 영수증 업로드 -> OCR 분류 -> 거래 자동 생성
- 카테고리 관리(기본 + 사용자)
- 월 예산 관리
- 대시보드 집계(월/카테고리/일별/연도/12개월)
- OCR 월 사용량 관리

### 3.3 ocr-ai-service
- 이미지 파일 OCR 텍스트 추출
- AI 분류 결과 반환(상호/금액/카테고리/신뢰도)

## 4. API 요약
### 4.1 Auth API (`:8001`)
- `POST /signup`
- `POST /login`
- `GET /me`
- `PUT /me/email`
- `PUT /me/password`
- `DELETE /me`
- `POST /billing/create-checkout-session`
- `POST /billing/sync-subscription`
- `POST /billing/cancel-subscription`
- `POST /billing/webhook`

### 4.2 Ledger API (`:8002`)
- Transactions: `POST/GET /transactions`, `GET /transactions/recent`, `GET /transactions/export/csv`, `GET/PUT/DELETE /transactions/{tx_id}`
- Receipts: `POST /receipts/upload`
- Budget: `POST /budget`, `GET /budget`
- Categories: `GET/POST /categories`, `PUT/DELETE /categories/{category_id}`
- Dashboard: `/dashboard/overview`, `/summary`, `/category`, `/daily`, `/ai-insight`, `/recent`, `/yearly-summary`, `/last-12-months`, `/transactions/{tx_id}`
- OCR usage: `GET /ocr/usage`

### 4.3 OCR API (`:8003`)
- `POST /ocr/classify`

## 5. DB 구조 요약
- `auth_db`: `users`
- `ledger_db`: `documents`, `transactions`, `categories`, `ocr_usage_monthly`, `budgets`

핵심 관계
- `transactions.document_id -> documents.document_id`
- `transactions.category_id -> categories.category_id`
- 대부분 테이블이 `user_id` 기준 멀티테넌트 분리

## 6. 배포/인프라 구조 요약
### 6.1 현재 배포 (deploy.yml 기준)
- GitHub Actions (`push main`)
- ECR 로그인/이미지 푸시
- EC2 SSH 접속 후 `docker compose pull && docker compose up -d`
- 현재 자동배포 대상: `ledger-service`
- 리전: `ap-northeast-2`

### 6.2 목표 인프라
- VPC + ECS Fargate + RDS + S3 + ALB + CloudFront + Route53

## 7. CI/CD 구조 요약
- 현재
  - CD 중심(배포 자동화)
  - 테스트/lint/security scan 단계 미포함
- 개선 방향
  - CI(테스트/정적분석) 추가
  - 이미지 태그 `latest + sha` 병행
  - 서비스별 분리 배포

## 8. 보안 구조 요약
- JWT Bearer 인증
- Stripe webhook signature 검증
- 시크릿은 GitHub Secrets/AWS Secrets Manager로 분리 권장
- IAM 최소권한, RDS 퍼블릭 차단, TLS 적용

## 9. 운영/모니터링 구조 요약
- CloudWatch 기반 메트릭/로그/알람
- 주요 지표: 5xx, p95 latency, ECS/RDS 자원 사용률
- Runbook 기반 장애 대응 + 롤백 절차 문서화
