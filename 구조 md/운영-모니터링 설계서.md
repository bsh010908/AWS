# 운영/모니터링 설계서

## 1. 목적
- 서비스 안정성을 수치로 관리한다(SLI/SLO).
- 장애를 조기 탐지하고 평균 복구시간(MTTR)을 단축한다.
- 로그/메트릭/트레이싱을 통합해 운영 가시성을 확보한다.

## 2. 운영 대상
- 애플리케이션: `auth-service`, `ledger-service`, `ocr-ai-service`
- 데이터: MySQL(RDS)
- 외부연동: Stripe, OCR 처리 흐름

## 3. 모니터링 아키텍처
- Metrics: CloudWatch Metrics
- Logs: CloudWatch Logs
- Alert: CloudWatch Alarm + SNS/Slack
- Trace: AWS X-Ray 또는 OpenTelemetry(권장)
- Dashboard: CloudWatch Dashboard + Grafana(선택)

## 4. 핵심 SLI/SLO
### 4.1 가용성
- SLI: 성공 응답 비율(2xx/3xx)
- SLO: 월 99.9% 이상

### 4.2 지연시간
- SLI: p95 응답시간
- SLO
  - auth API p95 < 300ms
  - ledger 일반 조회 p95 < 500ms
  - receipt upload p95 < 3s (OCR 제외), end-to-end < 10s

### 4.3 오류율
- SLI: 5xx 비율
- SLO: 분당 1% 이하

## 5. 수집 지표
### 5.1 애플리케이션
- 요청 수(RPS)
- 상태코드 분포(2xx/4xx/5xx)
- 라우터별 latency(p50/p95/p99)
- OCR 성공/실패 건수
- Stripe webhook 처리 성공률

### 5.2 인프라
- ECS: CPU/Memory 사용률, task restart 횟수
- ALB: TargetResponseTime, 5xx, HealthyHostCount
- RDS: CPU, 연결 수, FreeStorage, Read/Write latency

### 5.3 비즈니스
- 일/월 활성 사용자
- 거래 생성 건수
- OCR 사용량(`ocr_usage_monthly`)
- PRO 구독 전환율/해지율

## 6. 로그 설계
- 포맷
  - JSON 구조 로그 권장
  - 필수 필드: `timestamp`, `service`, `level`, `trace_id`, `user_id(optional)`, `path`, `status_code`, `latency_ms`
- 개인정보/민감정보
  - 비밀번호/토큰/카드정보 로그 금지
- 보관
  - 운영로그 30일(기본), 감사로그 90일 이상 권장

## 7. 알람 정책
### 7.1 즉시 알람(P1)
- ALB 5xx 급증(5분 평균 > 2%)
- ECS task 연속 비정상 종료
- RDS 연결 불가/CPU 과부하(> 90%, 5분)

### 7.2 일반 알람(P2)
- p95 latency 임계치 초과 10분 지속
- OCR 실패율 > 5%
- webhook 실패율 증가

### 7.3 통지 채널
- SNS -> Slack/Email
- 야간 온콜 담당자 로테이션

## 8. 장애 대응 Runbook
1. 알람 확인: 영향 범위/시점 파악
2. 상태 확인: ALB, ECS, RDS, 외부연동 상태 점검
3. 임시 조치: 스케일아웃/재시작/기능 플래그 차단
4. 복구 조치: 롤백 또는 핫픽스 배포
5. 사후 분석: RCA 문서 + 재발방지 액션

## 9. 운영 배포 정책
- 배포 시간대: 트래픽 저점 우선
- 배포 전 체크
  - 최근 알람 안정 상태
  - DB 마이그레이션 영향 검토
- 배포 후 검증(10~15분)
  - 오류율/latency/SLO 지표 관찰

## 10. 용량 관리
- ECS Auto Scaling 기준
  - CPU 60% 초과 5분 시 scale out
  - CPU 30% 미만 15분 시 scale in
- RDS 용량
  - 저장공간 80% 도달 전 증설 계획
- 피크 이벤트(월말 정산 등) 대비 사전 스케일링

## 11. 백업/복구 운영
- RDS 자동백업 + 일일 스냅샷
- 복구 목표
  - RPO: 15분
  - RTO: 1시간
- 분기 1회 복구 리허설
